## О проекте

Тренажёр предназначен для практического освоения методов тестирования программного обеспечения по методу «чёрного ящика». Студент получает описание функции и должен самостоятельно составить набор тест-кейсов, покрывающий все классы эквивалентности и граничные значения. После отправки система автоматически проверяет результаты, оценивает покрытие и даёт рекомендации по улучшению.

Вся логика работает на стороне клиента — эталонные реализации функций выполняются прямо в браузере. Никакой регистрации, базы данных или серверной части не требуется.

## Возможности

### Задания

6 задач трёх уровней сложности с эталонными реализациями:

| Задание | Сложность | Методы тестирования |
|---------|-----------|---------------------|
| Факториал | Легко | Классы эквивалентности, граничные значения |
| Високосный год | Легко | Логические условия, граничные значения |
| Простое число | Средне | Нелинейные классы, граничные значения |
| Калькулятор скидки | Средне | Многофакторное тестирование |
| Треугольник | Сложно | Комбинаторное тестирование, граничные значения |
| Валидация пароля | Сложно | Проверка форматов, комбинации нарушений |

### Тренажёр

- Добавление тест-кейсов с входными значениями, ожидаемым результатом, категорией и комментарием
- Поддержка функций с одним и несколькими параметрами
- Категоризация: нормальное значение, граничное значение, исключение, недопустимый тип
- Горячая клавиша `Ctrl+Enter` для быстрой проверки

### Автоматическая оценка

Система оценивает три аспекта:

- **Покрытие классов эквивалентности (40%)** — какие классы покрывают введённые тесты
- **Покрытие граничных значений (30%)** — какие граничные точки протестированы
- **Корректность ожиданий (30%)** — совпадают ли ожидаемые результаты с фактическими

### Результаты

- Анимированные круговые диаграммы с оценками по каждой метрике
- Общая оценка с текстовой градацией (Отлично / Хорошо / Удовлетворительно / Неудовлетворительно / Плохо)
- Детальная таблица сравнения ожидаемых и фактических результатов для каждого тест-кейса
- Рекомендации по улучшению покрытия
- Кнопка экспорта результатов в буфер обмена

### Дополнительно

- Тёмная и светлая тема с автоопределением системных настроек
- Сохранение прогресса и текущих сессий в `localStorage`
- Прогресс-бар выполнения заданий с лучшими оценками
- Обучающий раздел с теорией методов тестирования
- Toast-уведомления при действиях
- Адаптивный дизайн для мобильных устройств

## Быстрый старт

### Требования

- Node.js >= 18.17
- npm, yarn или bun

### Установка

```bash
git clone https://github.com/<ваш-username>/test-trainer.git
cd test-trainer
npm install
```

### Запуск

```bash
# Режим разработки (hot-reload)
npm run dev

# Сборка для продакшена
npm run build && npm run start
```

Приложение будет доступно на [http://localhost:3000](http://localhost:3000).

## Как работает оценка

Для каждого задания определены:

1. **Классы эквивалентности** — группы входных данных с одинаковым поведением функции
2. **Граничные значения** — точки на границах диапазонов, где вероятны ошибки
3. **Эталонная функция** — реализация, по которой проверяются ожидаемые результаты

При проверке каждый тест-кейс прогоняется через эталонную функцию, затем:

- Вычисляется покрытие классов эквивалентности на основе входных данных и результатов
- Вычисляется покрытие граничных значений путём сопоставления входов с эталонными граничными точками
- Ожидаемый результат сравнивается с фактическим (с поддержкой JSON, ошибок и русскоязычных значений)

## Добавление новых заданий

Чтобы добавить своё задание, добавьте новый объект в массив `tasks` в `src/lib/tasks.ts`:

```typescript
{
  id: 7,
  name: "Название",
  difficulty: "Средне",
  description: "Описание функции...",
  signature: "myFunction(x: number): string",
  topics: ["Классы эквивалентности"],
  params: [
    { name: "x", type: "number", description: "Описание параметра" }
  ],
  returnType: "string",
  code: `function myFunction(x: number): string { ... }`,
  equivalenceClasses: [
    { id: "ec1", name: "EC1: ...", description: "...", exampleValues: [1, 2] }
  ],
  boundaryValues: [
    { value: 0, description: "Нижняя граница" }
  ]
}
```

Затем добавьте эталонную функцию в объект `referenceFunctions` и хеурики покрытия в `src/lib/evaluator.ts`.
