## 📖 О проекте

Тренажёр предназначен для **практического освоения методов тестирования** программного обеспечения по принципу «чёрного ящика». Студент получает описание функции и должен самостоятельно составить набор тест-кейсов, покрывающий все классы эквивалентности и граничные значения. После отправки система автоматически проверяет результаты, оценивает покрытие и даёт рекомендации по улучшению.

### Почему этот тренажёр?

- 🎯 **Практическая направленность** — не просто теория, а реальные задания с автоматической проверкой
- 🧠 **Адаптивная сложность** — 6 задач от простых до продвинутых, от одного до трёх параметров
- ⚡ **Полностью клиентская** — вся логика работает в браузере, никаких серверов и баз данных
- 🔒 **Без регистрации** — открыл страницу и сразу работаешь, прогресс сохраняется автоматически
- 📊 **Детальная обратная связь** — не просто «правильно/неправильно», а конкретные рекомендации, что покрыто, а что нет

## ✨ Возможности

### 📋 Задания

6 задач трёх уровней сложности с эталонными реализациями, которые выполняются прямо в браузере:

| # | Задание | Сложность | Методы тестирования | Параметры |
|---|---------|-----------|---------------------|-----------|
| 1 | **Факториал** | 🟢 Легко | Классы эквивалентности, граничные значения | 1 (`n`) |
| 2 | **Високосный год** | 🟢 Легко | Логические условия, граничные значения | 1 (`year`) |
| 3 | **Простое число** | 🟡 Средне | Нелинейные классы, граничные значения | 1 (`n`) |
| 4 | **Калькулятор скидки** | 🟡 Средне | Многофакторное тестирование, граничные значения | 2 (`price`, `discount`) |
| 5 | **Треугольник** | 🔴 Сложно | Комбинаторное тестирование, граничные значения | 3 (`a`, `b`, `c`) |
| 6 | **Валидация пароля** | 🔴 Сложно | Проверка форматов, комбинации нарушений | 1 (`password`) |

### 🏋️ Тренажёр

Интерактивное рабочее пространство для создания тест-кейсов:

- **Форма добавления тестов** — входные значения (один или несколько параметров), ожидаемый результат, категория и комментарий
- **Категоризация тест-кейсов** — нормальное значение, граничное значение, исключение, недопустимый тип
- **Таблица тест-кейсов** — наглядный список всех добавленных тестов с возможностью удаления
- **Горячая клавиша** `Ctrl + Enter` / `Cmd + Enter` — мгновенная проверка без клика по кнопке
- **Автосохранение** — текущая сессия сохраняется в `localStorage`, можно закрыть и вернуться позже

### 📊 Автоматическая оценка

Система оценивает три аспекта с весовыми коэффициентами:

| Метрика | Вес | Описание |
|---------|-----|----------|
| **Покрытие классов эквивалентности** | **40%** | Какая доля классов покрыта введёнными тестами |
| **Покрытие граничных значений** | **30%** | Какие граничные точки протестированы |
| **Корректность ожиданий** | **30%** | Совпадают ли ожидаемые результаты с фактическими |

**Итоговая формула:** `EC × 0.4 + BV × 0.3 + Correctness × 0.3`

### 🏆 Результаты

Подробная аналитика после каждой проверки:

- **Анимированные круговые диаграммы** с оценками по каждой метрике (EC, BV, Корректность)
- **Общая оценка** с текстовой градацией:
  - 🌟 **Отлично** (≥ 90%)
  - 👍 **Хорошо** (≥ 75%)
  - 📝 **Удовлетворительно** (≥ 60%)
  - ⚠️ **Неудовлетворительно** (≥ 40%)
  - ❌ **Плохо** (< 40%)
- **Детальная таблица** сравнения ожидаемых и фактических результатов для каждого тест-кейса
- **Рекомендации** — список непокрытых классов эквивалентности и граничных значений с подсказками
- **Экспорт в буфер обмена** — форматированный текст с оценками и таблицей результатов

### 🎨 Интерфейс и UX

- 🌓 **Тёмная и светлая тема** с автоопределением системных настроек и переключателем
- 💾 **Сохранение прогресса** — лучшие результаты по каждому заданию хранятся в `localStorage`
- 📈 **Прогресс-бар** — визуальный индикатор выполненных заданий с цветовой градацией
- 📚 **Обучающий раздел** — теория методов тестирования с примерами и советами
- 🔔 **Toast-уведомления** — мгновенная обратная связь при добавлении/удалении тестов и проверке
- 📱 **Адаптивный дизайн** — корректная работа на мобильных устройствах и планшетах
- ✨ **Плавные анимации** — переходы между вкладками с помощью Framer Motion

## 🚀 Быстрый старт

### Требования

- **Node.js** >= 18.17
- **npm**, **yarn** или **bun**

### Установка

```bash
# Клонируйте репозиторий
git clone https://github.com/<your-username>/test-trainer.git
cd test-trainer

# Установите зависимости
npm install
```

### Запуск

```bash
# Режим разработки с hot-reload
npm run dev

# Сборка для продакшена
npm run build
npm run start
```

Приложение будет доступно на **[http://localhost:3000](http://localhost:3000)**.

> 💡 Также можно запустить через `bun` — просто замените `npm` на `bun` в командах выше.

## 🔍 Как работает оценка

Для каждого задания определены три компонента:

1. **Классы эквивалентности (EC)** — группы входных данных, на которых функция ведёт себя одинаково. Например, для факториала: положительные числа ≤ 20, отрицательные числа, нецелые числа и т.д.
2. **Граничные значения (BV)** — точки на границах диапазонов, где вероятность ошибки максимальна. Например: 0, 1, 20, 21 для факториала.
3. **Эталонная функция** — корректная реализация, по которой проверяются ожидаемые результаты студента.

### Процесс проверки

```
Тест-кейсы студента
       │
       ▼
┌───────────────────────┐
│ Выполнение через      │
│ эталонную функцию     │
└───────────┬───────────┘
            │
            ▼
┌───────────────────────┐   ┌─────────────────────┐
│ Покрытие EC (40%)     │   │ Покрытие BV (30%)   │
│ Сопоставление         │   │ Сопоставление       │
│ входов с примерами    │   │ входов с граничными │
│ + эвристики           │   │ значениями          │
└───────────┬───────────┘   └──────────┬──────────┘
            │                          │
            ▼                          ▼
┌───────────────────────┐   ┌─────────────────────┐
│ Корректность (30%)    │   │ Итоговая оценка     │
│ Сравнение ожидаемого  │   │ = EC×0.4 + BV×0.3   │
│ и фактического        │   │ + Correct×0.3       │
└───────────────────────┘   └─────────────────────┘
```

### Особенности сравнения результатов

- Поддержка **JSON-сравнения** для сложных возвращаемых значений (например, объект с полями `valid` и `errors`)
- **Чувствительность к регистру** отключена для строковых значений
- Поддержка **русскоязычных значений**: `да`/`нет` ↔ `true`/`false`
- **Сравнение ошибок** по подстроке — если ожидается «Ошибка», а функция выбрасывает «Ошибка: число слишком большое», это считается совпадением

## ➕ Добавление новых заданий

Добавьте новый объект в массив `tasks` в файле `src/lib/tasks.ts`:

```typescript
{
  id: 7,
  name: "Название задания",
  difficulty: "Средне", // "Легко" | "Средне" | "Сложно"
  description: "Описание функции, что она делает и какие возвращает значения...",
  signature: "myFunction(x: number, y: string): boolean",
  topics: ["Классы эквивалентности", "Граничные значения"],
  params: [
    { name: "x", type: "number", description: "Описание первого параметра" },
    { name: "y", type: "string", description: "Описание второго параметра" }
  ],
  returnType: "boolean",
  code: `function myFunction(x: number, y: string): boolean {
    // реализация функции
    return true;
  }`,
  equivalenceClasses: [
    {
      id: "ec1",
      name: "EC1: Нормальные значения",
      description: "Описание класса эквивалентности...",
      exampleValues: [1, 5, 10]
    }
  ],
  boundaryValues: [
    { value: 0, description: "Нижняя граница диапазона" },
    { value: 100, description: "Верхняя граница диапазона" }
  ]
}
```

Затем:

1. Добавьте **эталонную функцию** в объект `referenceFunctions` в том же файле
2. При необходимости добавьте **эвристики покрытия EC** в функцию `findCoveredEquivalenceClasses` в `src/lib/evaluator.ts`

## ❓ FAQ

### Требуется ли интернет для работы?

Нет! После первоначальной установки (`npm install`) тренажёр работает полностью оффлайн. Все эталонные функции выполняются в браузере.

### Где хранятся мои результаты?

Прогресс и текущие сессии сохраняются в `localStorage` браузера. При очистке данных браузера результаты будут сброшены.

### Можно ли использовать на мобильных устройствах?

Да, интерфейс адаптирован для мобильных устройств. Рекомендуемый минимальный размер экрана — 320px.

### Как сбросить прогресс?

Откройте консоль браузера (F12) и выполните:

```javascript
localStorage.removeItem('test-trainer-progress');
localStorage.removeItem('test-trainer-session-1'); // и т.д. для каждой сессии
location.reload();
```

### Как добавить свои задания?

См. раздел [Добавление новых заданий](#-добавление-новых-заданий) выше. Достаточно добавить описание и эталонную функцию в `src/lib/tasks.ts`.

### Почему оценка не 100%, хотя все тесты проходят?

Оценка учитывает не только корректность ожиданий, но и **покрытие** — покрытие классов эквивалентности (40%) и граничных значений (30%). Возможно, некоторые классы или граничные точки не покрыты вашими тестами. Посмотрите раздел «Рекомендации» в результатах.
